# Dense 7056 Auxiliary Geography Run

## Summary

The first spatially aware auxiliary geography feature set did not improve the
best dense-grid weather model on the existing random reservoir validation split.

The fair comparison is the 16M/1M reservoir split using the same residual MLP
shape and the same NASA POWER dense 7,056-location dataset.

| Run | Inputs | Params | Train / Val Rows | Best Epoch | Best Pinball | GHI MAE | DNI MAE | DHI MAE | Temp MAE | Wind MAE |
| --- | ---: | ---: | ---: | ---: | ---: | ---: | ---: | ---: | ---: | ---: |
| `dense_7056_16m_residual_384x6` | 64 | 1,810,191 | 16M / 1M | 18 | 8.736989 | 36.047 | 84.169 | 21.731 | 1.428 | 1.497 |
| `dense_7056_16m_geo_residual_384x6` | 278 | 1,892,367 | 16M / 1M | 20 | 8.814697 | 36.483 | 84.818 | 21.914 | 1.454 | 1.492 |

The geo model slightly improved wind MAE, but solar irradiance and temperature
were worse enough that aggregate pinball loss regressed.

## Auxiliary Feature Set

The geography table is keyed by `location_id` and generated by
`collect_aux_geography.py` from two coarse global rasters:

- NOAA ETOPO 2022 60 arc-second relief
- ESA CCI/C3S 2020 300 m land cover

The model excludes identifier and duplicate coordinate columns from the aux CSV,
then normalizes the remaining columns across locations:

- included aux columns: 214
- base weather/time/coordinate features: 64
- total model inputs: 278

The features are spatial aggregates rather than only point features:

- 25 km, 100 km, and 250 km radii
- eight compass sectors for directional context
- elevation/bathymetry statistics and water fractions
- land-cover fractions and selected sector fractions

## Cache Shortcut

The slow path is rebuilding a reservoir sample by scanning the full normalized
NASA gzip CSV. That costs tens of minutes because the file has 309,391,488 rows.

For static per-location features, the faster path is to reuse an existing base
sample cache and append normalized aux features from the cached lat/lon columns:

```sh
python3 scripts/augment_weather_cache_with_aux.py \
  --base-cache cache/dense_7056/samples_reservoir_train16000000_val1000000_ts2_vs2_v2.npz \
  --aux-features data/location_features_v1.csv \
  --out cache/dense_7056_geo/samples_reservoir_train16000000_val1000000_ts2_vs2_aux87a6a86840e4_v3.npz \
  --train-limit 16000000 \
  --val-limit 1000000 \
  --chunk-size 500000
```

This built the full aux cache in well under a minute on the RTX host. The full
cache size was 18 GiB.

## Training Command

```sh
PYTHONUNBUFFERED=1 python scripts/train_weather_mlp_torch.py \
  --data data/nasa_power_hourly_global_grid_7056.csv.gz \
  --out-dir results/dense_7056_16m_geo_residual_384x6 \
  --cache-dir cache/dense_7056_geo \
  --sample-mode reservoir \
  --train-limit 16000000 \
  --val-limit 1000000 \
  --train-stride 2 \
  --val-stride 2 \
  --aux-features data/location_features_v1.csv \
  --architecture residual-mlp \
  --residual-width 384 \
  --residual-blocks 6 \
  --residual-scale 0.5 \
  --epochs 20 \
  --batch-size 32768 \
  --learning-rate 0.001 \
  --resident-device
```

## Interpretation

This result is a negative result for this particular tabular feature shape and
validation split. It does not prove geography is unhelpful. The current split is
a random reservoir over the dense global grid, so nearby locations and very
similar geography can appear in both train and validation samples. A better test
for geography should use held-out spatial regions or interpolation/extrapolation
splits.

Recommended next experiments:

- compare feature subsets: point-only, aggregate-only, sector-only, land-cover-only, relief-only
- add stronger regularization or a smaller geo branch before concatenation
- evaluate on region-held-out validation splits
- try raster patches only after scalar/sector features are understood
